# Portfolio redesign — design spec

Date: 2026-07-03

## Goal

Replace the current single-scroll portfolio (`index.html`) with a cleaner, better-organized
site that separates content into focused pages, gives publications real visual hierarchy, and
uses a warmer, more distinctive visual style than the current generic light-blue/dark-mode theme.

## Workflow

- All work happens on branch `redesign-2026` in this repo (already created). `main` (the branch
  GitHub Pages serves from) is untouched until the redesign is reviewed and explicitly approved.
- Nothing gets pushed to `origin` until sign-off. Local commits on the branch are fine throughout.
- Old unused files (`index-modern.html`, `index-backup.html`, `script-backup.js`,
  `style-backup.css`, `script-modern.js`, `style-modern.css`) are deleted as part of this work —
  they're dead variants from earlier redesign attempts, not needed going forward.

## Site structure

Hybrid multi-page site with a persistent top nav and footer, five pages:

| Page | File | Content |
|---|---|---|
| Home | `index.html` | Hero photo, name, one-line "what I'm doing now", CTA buttons to Publications and CV |
| Research | `research.html` | Research interests (from CV), current activity photos (added later) |
| Education | `education.html` | Academic timeline: UF PhD, University of St. La Salle B.Sc., with institution logos |
| Publications | `publications.html` | Software/Projects, then Journal Articles / Conference Proceedings / Preprints, each grouped by type and newest-first (mirrors the CV) |
| News & Media | `news.html` | Photo gallery: conferences, awards, meetings |

Full CV detail (presentations, teaching, memberships, awards list, skills) is **not** duplicated
across every page — the site stays focused on the highlights, and the footer's CV link is the
canonical exhaustive record. This is a deliberate scope cut in service of "clean, uncluttered."

### Navigation & responsive behavior

- Top horizontal tab bar: Home · Research · Education · Publications · News & Media.
- Collapses to the existing hamburger/overlay pattern already implemented in `script.js`
  (`nav__toggle`) below 768px — reused, not rebuilt, so mobile/tablet/desktop all work from one
  pattern.
- Footer (CV download link + contact info) renders on every page via shared partial (see
  Implementation notes).

## Visual system

**Palette** ("warm academic" — cream base + light-blue cards):

| Token | Hex | Use |
|---|---|---|
| `--bg-page` | `#FAF6EC` | Page background (cream) |
| `--bg-card` | `#DCEBF7` | Cards, tags, highlighted sections (light blue) |
| `--accent` | `#2C5C82` | Headings, links, nav active state, buttons |
| `--text-primary` | `#3D3A30` | Body text (warm dark brown-gray, not pure black) |
| `--text-secondary` | `#6B6558` | Supporting text, captions |
| `--border` | `#E4DCC8` | Hairline borders, dividers (warm tan) |

Publication-type tags reuse the same family at low saturation so they read as related, not
random: Journal = blue tag, Conference = tan/khaki tag, Preprint = muted gray tag — each a pale
fill with dark text of the same hue (not black-on-color).

**Typography**: serif headings (Lora, via Google Fonts) paired with the existing sans-serif body
font (Inter). The serif is what gives the "warm academic" feel instead of a generic
tech/corporate look; Inter keeps body text easy to read at small sizes.

**Dark mode**: dropped for this redesign. The current toggle exists but the new palette is
purpose-built as a single warm light theme; a forced dark variant wasn't designed for it. Can
revisit as a future enhancement if wanted — flagging this as a scope cut, not an oversight.

## Implementation notes

- Stays vanilla HTML/CSS/JS — no build step, no framework — consistent with the current site and
  simplest for GitHub Pages.
- Shared header (nav) and footer are injected via a small `partials.js` (template strings into
  placeholder `<div id="site-header"></div>` / `<div id="site-footer"></div>`, not `fetch()` of
  `.html` partials) so the site still works when opened directly as a local file (`file://`)
  without a dev server, and so nav/footer edits happen in one place instead of five.
- CV footer link reuses the existing hosted PDF: `https://angelodecastro.github.io/angelo_cv/CV_angelo.pdf`.

## Out of scope (this pass)

- Dark mode support for the new palette.
- Research-tab and News & Media photos beyond the four already provided.
- Any change to `CV_Angelo_updated/main.tex` (handled separately).
- Analytics, comments, blog/CMS functionality — none of that exists today and isn't being added.
